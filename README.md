## Why OpenMEP?

Commercial MEP tools cost thousands per seat and lock results in proprietary formats. OpenMEP is built on three principles:

- **Region-aware** вҖ” design codes switch automatically: BS 7671 for GCC/UK, IS 3961 for India, AS/NZS 3008 for Australia. Three-level selector: Region вҶ’ Country/State вҶ’ Utility/Authority.
- **Standard-cited** вҖ” every result references the exact clause and table used. Full numerical tables (ampacity, correction factors, pressure drop, etc.) are embedded directly in the codebase вҖ” no external lookups.
- **Audit-ready** вҖ” one-click PDF reports with letterhead, step-by-step calculation workings, and an engineer sign-off block.

## Platform Features

Seven features that sit across all calculation modules, turning individual results into a complete project workflow.

### 1 В· Project Workspace вҖ” Single Source of Truth

Store your building data once: project name, region, number of floors, space types, and design conditions. Every calculation module reads from the workspace automatically вҖ” change the region or ambient temperature and all 26 modules update without re-entering data.

- **REST API:** `POST /api/projects` В· `GET /api/projects/{id}` В· `PUT /api/projects/{id}`
- **Streamlit:** *Project Workspace* page вҖ” fill in once, use everywhere
- Persists between sessions; JSON export for archiving

### 2 В· Submission Packager

One click generates a ready-to-send ZIP archive containing:
- All calculation PDFs for the project
- Cover letter (auto-populated with project metadata)
- Cross-module compliance matrix
- BOQ summary sheet
- Version log (who ran what and when)

```bash
curl -X POST http://localhost:8080/api/submission/package \
  -H "Content-Type: application/json" \
  -d '{"projectId": "proj_abc123", "includeModules": ["electrical","mechanical","fire"]}'
# Returns a downloadable ZIP
```

### 3 В· Compliance Guardian

Runs automatically before every submission package is built. Checks every module result against the regional authority limits and flags any breach with the exact standard clause.

| Check | Example rule |
|-------|-------------|
| Voltage drop | DEWA В§4.3.2 вҖ” max 2.5 % for final circuits |
| Cable temperature | IS 732 В§6.4 вҖ” ambient derating mandatory above 40 В°C |
| Sprinkler spacing | NFPA 13 В§8.6.3 вҖ” max 4.6 m centre-to-centre |
| Pipe velocity | AS/NZS 3500 В§3.4 вҖ” max 3 m/s cold water |

Returns structured JSON with `PASS`, `WARN`, and `FAIL` per module вҖ” the Streamlit UI shows a colour-coded summary.

```bash
curl -X POST http://localhost:8080/api/submission/compliance-check \
  -H "Content-Type: application/json" \
  -d '{"moduleResults": [{"module":"electrical","region":"DEWA","parameters":{"voltage_drop_percent":3.1}}]}'
```

### 4 В· BIM / IFC & CSV Bridge

**Import:** Upload an IFC file or a Revit schedule CSV вҖ” the bridge extracts room data, floor counts, space categories, and design loads and auto-populates the Project Workspace. No manual re-entry from the BIM model.

**Export:** Download calculated results as:
- CSV (column-per-parameter, ready for Excel or Revit lookup tables)
- IFC property sets (`.ifc`) ready to import back into Revit or ArchiCAD

```bash
# Import from IFC (Node.js API вҖ” port 8080)
curl -X POST http://localhost:8080/api/bim/import \
  -F "file=@building_model.ifc" \
  -F "projectId=proj_abc123"

# Export results as CSV
curl "http://localhost:8080/api/bim/export/csv?projectId=proj_abc123&module=electrical"

# Export as IFC property sets
curl "http://localhost:8080/api/bim/export/ifc?projectId=proj_abc123"
```

### 5 В· Value Engineering & Cost Optimizer

After every calculation, the optimizer scans compliant alternatives and ranks them by estimated cost saving in the local currency (AED, INR, GBP, AUD). Every suggestion is rule-based and cites the standard that makes it valid вҖ” no AI guessing.

Example output for a cable sizing result:

| Option | Cable | Saving (AED) | Standard basis |
|--------|-------|-------------|----------------|
| Current | 70 mmВІ XLPE/Cu | вҖ” | BS 7671 Table 4D5A |
| **Option A** | 50 mmВІ XLPE/Al | **вҲ’18%** | BS 7671 В§523.6 вҖ” Al permitted for вүҘ16 mmВІ |
| Option B | 70 mmВІ XLPE/Al | вҲ’11% | BS 7671 В§523.6 |

```bash
curl -X POST http://localhost:8080/api/optimize/electrical \
  -H "Content-Type: application/json" \
  -d '{"projectId":"proj_abc123","region":"gcc","load_kw":45,"cable_type":"XLPE_CU"}'
```

### 6 В· Project Version History

Full audit trail of every project configuration state. Each save captures project metadata, engineering parameters, compliance status, and a BOQ snapshot with cost deltas (e.g. _BOQ +AED 124,000_ between versions). Includes a diff viewer comparing any two versions.

- **REST API:** `POST /api/projects/{id}/versions` В· `GET /api/projects/{id}/versions`
- **Streamlit:** *Version History* page (page 27)
- Version numbers are sequential and append-only вҖ” no overwriting

### 7 В· Company Branding & Report Templates

Firms store a logo, stamp image, primary colour hex code, and footer text once per project. Every PDF calculation report вҖ” cable sizing, cooling load, fire protection, and the rest вҖ” auto-injects the branding (branded letterhead, stamp block, coloured headings, custom footer). Custom report templates let you define per-client header/footer overrides.

- **REST API:** `POST/GET /api/projects/{id}/branding` В· `POST/GET /api/projects/{id}/templates`
- **Streamlit:** *Company Branding* (page 28) В· *Report Templates* (page 29)
- Supports logo, letterhead, and stamp as base64-encoded images

## Security

### API Key Authentication

The API supports optional `X-API-Key` authentication. Set the `API_KEY` environment variable to require the header on all calculation endpoints. Health check and documentation endpoints remain public.

```bash
# Generate a key and add to .env
echo "API_KEY=$(python -c "import secrets; print(secrets.token_urlsafe(32))")" >> .env

# Authenticated request:
curl -X POST http://localhost:8000/api/electrical/cable-sizing \\
  -H "X-API-Key: $API_KEY" \\
  -H "Content-Type: application/json" \\
  -d '{"region":"gcc","load_kw":45,...}'

# /health and /docs are always public вҖ” no key required
```

When `API_KEY` is **not** set (the default), the API is open вҖ” appropriate for local development and deployments on private networks.

### Container Security

Docker containers run as a non-root user (`appuser`). Each service in `docker-compose.yml` runs a single process вҖ” no `sh -c "... & ..."` anti-patterns.

See [SECURITY.md](SECURITY.md) for the full disclosure policy.
